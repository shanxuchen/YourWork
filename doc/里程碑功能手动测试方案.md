## 2. 功能测试

### 2.1 测试一：里程碑行动项管理

**测试目标**：验证里程碑行动项的增删改查功能

#### 测试步骤

**步骤 1.1：进入项目详情页**
```
操作：访问 http://localhost:8001
     使用 admin / admin123 登录
     在首页点击任意项目名称进入项目详情页

预期结果：
- 显示项目详情页面
- 有三个Tab：里程碑、产出物、成员
- 页面顶部显示"项目详情"标题
- 右上角有"返回列表"和"编辑项目"按钮
```

---

**步骤 1.2：查看里程碑列表**
```
操作：点击"里程碑"Tab（默认选中）
     查看里程碑列表

预期结果：
- 显示里程碑列表
- 每个里程碑卡片显示：
  * 里程碑名称
  * 状态标签（已创建/进行中/已完成/已挂起）
  * 四个按钮：管理行动项、编辑依赖、上传产出物
  * 行动项进度（如："行动项进度: 0/0 已完成"）
  * 前置依赖（如："前置依赖: 无"）
```

---

**步骤 1.3：打开行动项管理对话框**
```
操作：找到任意一个里程碑
     点击该里程碑的"管理行动项"按钮

预期结果：
- 弹出"里程碑行动项 - [里程碑名称]"对话框
- 对话框居中显示
- 可以通过拖动顶部栏移动对话框
- 对话框内显示：
  * 当前行动项列表（如果有）
  * 底部有"+ 添加行动项"按钮
  * 底部有"取消"和"保存"按钮
```

---

**步骤 1.4：添加行动项**
```
操作：在行动项管理对话框中
     点击"+ 添加行动项"按钮
     在弹出的提示框中输入：完成需求文档
     点击"确定"

预期结果：
- 自动关闭提示框
- 对话框中新增一行行动项：
  * 复选框（未选中）
  * 文本输入框（显示"完成需求文档"）
  * 状态下拉框（默认"待处理"）
  * "×"删除按钮
- 选项：pending（待处理）、in_progress（进行中）、completed（已完成）
```

---

**步骤 1.5：继续添加更多行动项**
```
操作：再次点击"+ 添加行动项"按钮
     输入：设计系统架构
     点击"确定"
     重复步骤，添加：编写代码

预期结果：
- 对话框中显示 3 个行动项
- 每个行动项都有编辑、删除控件
- 列表可滚动（如果超出对话框高度）
```

---

**步骤 1.6：修改行动项状态**
```
操作：方法A - 勾选第1个行动项的复选框
     方法B - 在状态下拉框中选择"已完成"

预期结果：
- 该行动项的状态变为"已完成"
- 复选框被选中
- 状态标签颜色改变
```

---

**步骤 1.7：编辑行动项标题**
```
操作：点击第2个行动项的文本输入框
     修改标题为：设计系统架构 v1.0
     按回车键或点击其他地方

预期结果：
- 标题已更新
- 输入框失去焦点
```

---

**步骤 1.8：保存并验证**
```
操作：点击对话框底部的"保存"按钮

预期结果：
- 显示"保存成功"提示（绿色消息）
- 对话框自动关闭
- 里程碑列表中该里程碑的"行动项进度"更新为"1/3 已完成"
```

---

**步骤 1.9：验证行动项持久化**
```
操作：刷新页面（按F5或浏览器刷新按钮）
     再次点击同一里程碑的"管理行动项"按钮

预期结果：
- 对话框重新打开
- 显示之前添加的 3 个行动项
- 第1个状态为"已完成"，其余为"待处理"
- 第2个标题显示为"设计系统架构 v1.0"
- 数据已持久化到数据库
```

---

**步骤 1.10：删除行动项**
```
操作：找到第3个行动项（"编写代码"）
     点击该行右侧的红色"×"按钮

预期结果：
- 行动项被立即删除
- 对话框中剩余 2 个行动项
- 不需要点击保存即可删除
```

---

**步骤 1.11：取消修改**
```
操作：修改任意行动项（但不保存）
     点击对话框底部的"取消"按钮

预期结果：
- 对话框关闭
- 未保存的修改被丢弃
- 行动项列表保持原样
```

### 2.2 测试二：依赖关系管理

**测试目标**：验证里程碑前置依赖的管理功能

#### 测试步骤

**步骤 2.1：打开依赖编辑对话框**
```
操作：在项目详情页的里程碑Tab中
     找到一个状态为"已创建"或"进行中"的里程碑
     点击该里程碑的"编辑依赖"按钮

预期结果：
- 弹出"编辑前置依赖 - [里程碑名称]"对话框
- 显示当前项目的所有可选里程碑
- 每个里程碑前有复选框
- 显示每个里程碑的状态
- 显示复选框列表的标题："选择此里程碑的前置里程碑"
- 底部有"取消"和"保存"按钮
```

---

**步骤 2.2：添加前置依赖**
```
操作：找到 1-2 个已完成或进行中的里程碑
     勾选这些里程碑的复选框
     点击"保存"按钮

预期结果：
- 显示"成功添加 N 个依赖关系"消息
- 对话框关闭
- 里程碑列表中该里程碑的"前置依赖"已更新
- 例如显示："前置依赖: 需求分析, 系统设计"
- 被依赖的里程碑在状态变更时会受到限制
```

---

**步骤 2.3：查看依赖详情**
```
操作：再次点击同一里程碑的"编辑依赖"按钮

预期结果：
- 对话框重新打开
- 之前选择的里程碑仍被选中（复选框有勾选标记）
- 可以继续修改依赖关系
```

---

**步骤 2.4：删除依赖**
```
操作：取消选择某个已选择的里程碑复选框
     点击"保存"按钮

预期结果：
- 显示"成功删除 N 个依赖关系"消息
- 该里程碑的前置依赖已更新
- 如果删除所有依赖，显示"前置依赖: 无"
```

---

**步骤 2.5：测试目标必须有依赖的约束**
```
操作：1. 点击右上角"添加里程碑"按钮
     2. 输入名称：测试目标
     3. 类型选择：目标（如果有类型选项）或保持默认
     4. 点击"创建"
     5. 找到刚创建的"测试目标"
     6. 点击"编辑依赖"按钮
     7. 不选择任何前置依赖
     8. 点击"保存"按钮

预期结果：
- 显示错误提示："目标（验收点）必须有至少1个前置里程碑"
- 对话框不关闭
- 约束条件：deliverable 类型的里程碑必须有 1+ 个前置依赖
```

---

**步骤 2.6：测试里程碑可以有0个前置依赖**
```
操作：创建或选择一个类型为"里程碑"（非目标）的里程碑
     点击"编辑依赖"按钮
     不选择任何前置依赖
     点击"保存"按钮

预期结果：
- 保存成功
- 里程碑保持无前置依赖状态
- 前置依赖显示："前置依赖: 无"
```

### 2.3 测试三：状态变更验证

**测试目标**：验证状态变更时的前置条件检查

#### 测试步骤

**步骤 3.1：创建测试里程碑**
```
操作：准备两个里程碑用于测试
     里程碑A（前置）
     里程碑B（依赖）

如没有，创建方法：
     1. 点击"添加里程碑"按钮
     2. 输入名称：里程碑A
     3. 点击"创建"
     4. 输入名称：里程碑B
     5. 点击"创建"
```

---

**步骤 3.2：设置依赖关系**
```
操作：点击里程碑B的"编辑依赖"按钮
     选择里程碑A作为前置依赖
     点击"保存"

预期结果：
- 依赖关系创建成功
- 里程碑B显示："前置依赖: 里程碑A"
```

---

**步骤 3.3：尝试未完成前置依赖就启动（应该失败）**
```
操作：确认里程碑A的状态是"已创建"（未完成）
     点击里程碑B的状态标签
     或点击里程碑B的"编辑"按钮
     将状态改为"进行中"
     点击"保存"

预期结果：
- 显示错误提示："前置里程碑「里程碑A」尚未完成"
- 或类似："状态变更失败"
- 错误列表中列出所有未完成的前置里程碑
- 状态变更失败，里程碑B保持原状态
```

---

**步骤 3.4：完成前置依赖后启动（应该成功）**
```
操作：1. 点击里程碑A的"编辑"按钮或状态标签
     2. 将状态改为"已完成"
     3. 点击"保存"

     4. 现在再次修改里程碑B：
     点击里程碑B的"编辑"按钮或状态标签
     将状态改为"进行中"
     点击"保存"

预期结果：
- 里程碑A状态变为"已完成"（绿色标签）
- 里程碑B状态变更成功
- 里程碑B状态变为"进行中"（蓝色标签）
- 无错误提示
```

---

**步骤 3.5：测试依赖链**
```
操作：创建三个里程碑A、B、C
     设置依赖：B 依赖 A，C 依赖 B
     保持A为未完成状态
     尝试将C改为"进行中"

预期结果：
- 显示错误提示：包含所有未完成的前置依赖
- 错误中应该提到"里程碑B"和"里程碑A"
```
