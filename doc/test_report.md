## 🐛 Bug修复记录

### Bug #1: 数据库未初始化
**错误**: `sqlite3.OperationalError: no such table: users`
**影响**: 所有API调用失败
**修复**: 运行 `python init_db.py` 初始化数据库
**状态**: ✅ 已修复

### Bug #2: 里程碑列表测试代码错误
**错误**: `'list' object has no attribute 'get'`
**位置**: `test/automated_test_suite.py:315`
**原因**: API返回 `{"code": 0, "data": milestones}` 但测试期望 `data["data"]["milestones"]`
**修复**: 修改为 `data.get("data", [])`
**状态**: ✅ 已修复

### Bug #3: 测试用户缺失
**错误**: manager和worker用户无法登录
**原因**: 数据库中只有admin用户
**修复**: 在 `init_db.py` 中添加 manager 和 worker 用户
**状态**: ✅ 已修复

### Bug #4: 里程碑更新 NOT NULL 约束错误
**错误**: `sqlite3.IntegrityError: NOT NULL constraint failed: milestones.name`
**位置**: `main.py:872`
**原因**: 更新时传入 None 值给 NOT NULL 字段
**修复**: 修改为动态SQL，只更新提供的字段
**状态**: ✅ 已修复

### Bug #5: WebSocket URL错误
**错误**: WebSocket 403 Forbidden
**原因**: 测试使用 `/api/v1/ws` 但端点在 `/ws`
**修复**: 修改测试代码使用正确的URL
**状态**: ✅ 已修复

### Bug #6: 文件权限测试逻辑错误
**错误**: worker可以从非成员项目下载文件
**原因**: 测试中worker已被添加为项目成员
**修复**: 创建独立项目测试非成员访问
**状态**: ✅ 已修复

### Bug #7: 未授权访问测试cookie污染
**错误**: 测试session包含认证cookie
**原因**: 复用已登录的session
**修复**: 使用干净的session测试未授权访问
**状态**: ✅ 已修复
